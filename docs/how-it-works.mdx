## ‚ö° Request & Payment Flow

Here‚Äôs the step-by-step process for making a paid API/SDK request:

---
### 1Ô∏è‚É£ Make a Request

Send a request via SDK or REST API:

```bash

-H "Content-Type: application/json" \
-H "X-Network: solana" \
```

---
### 2Ô∏è‚É£ Payment Required (HTTP 402)

If payment is required, the server responds with `402 Payment Required` and x402 instructions.

The response body follows the **x402 Protocol** specification and looks like this:

```json
{
  "x402Version": 1,
  "error": "X-PAYMENT header is required",
  "accepts": [
    {
      "scheme": "exact",
      "network": "solana-devnet",
      "maxAmountRequired": "40000",
      "resource": "http://api.oryntai.xyz/api/v1/ai/images",
      "description": "Generate images",
      "mimeType": "",
      "payTo": "CEBAqJoQXomRdMubgTWL1fa99d2zr2rFTMPukk62fSUw",
      "maxTimeoutSeconds": 60,
      "asset": "4zMMC9srt5Ri5X14GAgXhaHii3GnPAEERYPJgZJDncDU",
      "outputSchema": {
        "input": {
          "type": "http",
          "method": "POST",
          "discoverable": true
        }
      },
      "extra": {
        "feePayer": "2wKupLR9q6wXYppw8Gr2NvWxKBUqm4PPJKkQfoxHDBg4"
      }
    }
  ]
}
```
This tells the client exactly **how to pay** for the request.

Key fields include:

- **`network`** ‚Äî which chain to use (`solana-devnet`, `base`, `polygon`, etc.)
- **`asset`** ‚Äî the token used for payment (e.g., USDC mint address)
- **`maxAmountRequired`** ‚Äî the maximum amount required to fulfill the request
- **`payTo`** ‚Äî wallet address to send the payment to
- **`maxTimeoutSeconds`** ‚Äî how long the payment window stays valid
- **`resource`** ‚Äî the specific endpoint being billed
- **`extra.feePayer`** ‚Äî optional facilitator address used for covering transaction fees

---
### 3Ô∏è‚É£ Pay & Resend

Once you receive the `402 Payment Required` response, your client (or wallet) performs a payment on the specified network, for example, sending USDC on Solana to the `payTo` address provided.

After the transaction is confirmed, you resend the **same original request**, but now include the `X-PAYMENT` header containing the on-chain transaction signature (proof of payment).

```bash
-H "X-PAYMENT: <your_payment_signature>"
```
---
### 4Ô∏è‚É£ Verification & Response

If the payment proof is valid and verified by the facilitator (via **PayAI**), Orynt will process your request and return the result (e.g., an image URL).

If the payment is **invalid** or **expired**, you‚Äôll receive another `402` response with updated payment instructions.
